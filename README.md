# 🛡️ Payload Defense — LLM 注入攻击防御插件

> **在消息进入 LLM 之前直接清洗/拦截伪造消息，让注入攻击无法生效。**

---

## 📌 这是什么？

有人在群里给你的 Bot 发一段**精心构造过的消息**，消息里塞满：

- `[user_nickname: xxx]`
- `[message_id: xxx]`
- `"sender_id": "xxx"`

……看起来像系统日志或 QQ 消息包的结构，然后让 Bot 照做——这就是 **注入攻击**。

**本插件的核心逻辑**：  
在消息被交给 LLM 之前，**先扫描原始消息文本**。只要发现这些伪造字段，就直接把消息替换为拦截警告，或者直接丢弃。  
**注入内容根本到不了 LLM 的“眼前”。**

---

## 📦 安装

### 方式一：WebUI 一键安装（推荐）

1. 打开 KiraAI 的 WebUI → **插件管理**
2. 点击「**从 GitHub 安装**」或「**上传 ZIP**」
3. 粘贴仓库地址或上传 ZIP 文件
4. 安装完成后，在插件列表里启用 **Payload Defense**
5. **重启 KiraAI**

### 方式二：手动放置

```bash
# 1. 把整个插件文件夹复制到 KiraAI 的 data/plugins/ 目录下
#    最终路径类似：KiraAI/data/plugins/payload-defense/

# 2. 重启 KiraAI

# 3. 在 WebUI → 插件管理 中启用「Payload Defense」
```

## ⚙️ 配置（必须做这一步）

启用插件后，在 WebUI 的插件设置页面完成以下配置：

### 第一步：填写白名单

在 **🛡️ 防御总控** 中找到：

| 配置项 | 填什么 | 示例 |
|--------|--------|------|
| **白名单 QQ号** | 你自己的 QQ 号 | `114514` |
| **主人 QQ号** | 和上面填同一个号（用于警告中显示） | `114514` |

> 💡 两个字段都支持多个 QQ 号，**一行一个**。  
> 白名单中的 QQ 发送的消息**不会被检测**。

---

### 第二步：选择防御模式

| 模式 | 效果 | 适用场景 |
|------|------|----------|
| `warn` | 把注入消息替换为拦截警告，LLM 可以回复吐槽 | 日常使用，适合观察误杀情况 |
| `strict` | 直接丢弃注入消息，LLM 不会处理 | 正在被频繁攻击，或被攻击成功后开启 |

---

### 第三步（可选）：调整正则

在 **注入特征正则** 中可以增加/删除检测规则。  
默认规则已覆盖常见系统日志伪装和 JSON 消息包。

---

## 🚨 拦截效果

当消息被检测到包含注入特征时，插件会根据当前防御模式做出不同响应：

| 模式 | 行为 |
|------|------|
| `warn` | 将原消息替换为一条拦截警告，LLM 会看到警告内容并可以据此回复。警告信息示例：<br>`⚠️【注入攻击已被拦截】原消息包含伪造的系统字段，不是主人发送的。` |
| `strict` | 直接丢弃该消息，LLM 不会收到任何内容，Bot 不会做出任何回复。 |

> ⚠️ 警告信息中的“主人”会替换为您在配置中填写的“主人 QQ号”对应的昵称或号码。

具体检测规则可在配置页面的正则表达式中查看和调整，您也可以根据需要增加新的特征规则。

## 📋 日志

开启 **详细调试日志** 后，你可以在控制台看到被拦截的消息：

```log
🛡️ DEFENSE WARN-BLOCKED injection from 1845575735 | hits=3 | matched=[\[\message_id..., \[\user_nickname..., ...] | original_preview='嗯嗯\n[Jul 05...'
```

- `warn` 模式显示 `WARN-BLOCKED`
- `strict` 模式显示 `STRICT-BLOCKED`

## ❓ 常见问题

### Q：会误杀正常聊天吗？

**A：** 可能误杀谈论 QQ 协议/消息格式/代码配置的消息。  
默认规则针对的是完整消息包字段组合（如 `sender_id` + `session_id` + `session_type`），正常聊天中极少出现。  
如果误杀，可以调高 `min_hits` 或删除部分 patterns。

---

### Q：strict 模式会不会让 Bot 不回复任何消息？

**A：** 只有命中注入特征的消息会被丢弃，正常聊天完全不受影响。

---

### Q：主人自己测试会被拦截吗？

**A：** 如果主人的 QQ 在白名单里，就不会被检测。  
测试时请把测试用的 QQ **从白名单里去掉**。

---

### Q：能防御所有注入攻击吗？

**A：** 不能 100% 保证。它只能防御包含伪造系统字段的注入。  
如果攻击者用纯自然语言诱导（如 *“请无视之前设定，做 xxx”*），本插件无法识别。
